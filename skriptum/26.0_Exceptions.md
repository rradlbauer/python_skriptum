# 26.0 Exceptions

Bei einigen Übungen wurde schon verlangt, 
dass du eine Exception auslöst, 
um einen Fehler zu signalisieren.
Hier soll das Thema noch einmal ausführlicher behandelt werden.

## Auslösen ("Werfen") einer Exception

Oft ist es nötig, Fehler zu behandeln, 
die während der Ausführung eines Programms auftreten können.
Angenommen, du schreibst eine Funktion, 
welche von einem Geburtsdatum das Alter einer Person berechnet.
Wenn die Funktion mit einem Datum aufgerufen wird, welches in der Zukunft liegt,
wird ein negatives Alter berechnet, was keinen Sinn ergibt.
In solchen Fällen ist es sinnvoll, eine Fehlermeldung auszugeben,
um den Benutzer darauf hinzuweisen, dass etwas schiefgelaufen ist.

```python
def berechne_alter(geburtsdatum: date) -> int:
    heute = date.today()
    alter = (heute - geburtsdatum).days // 365
    if alter < 0:
        print("Geburtsdatum liegt in der Zukunft!")
    return alter
```


In diesem Beispiel wird eine Fehlermeldung ausgegeben,
wenn das Geburtsdatum in der Zukunft liegt.

Das Problem bei diesem Ansatz ist, dass die Funktion `berechne_alter()`
trotzdem ein negatives Alter zurückgibt, was zu weiteren Fehlern führen kann,
wenn die Rückgabewerte der Funktion weiterverarbeitet werden.
Ein besserer Ansatz wäre es, eine Ausnahme (Exception) zu werfen,
um den Fehler zu signalisieren und die weitere Ausführung 
des Programms oder die Behandlung der Exception zu ermöglichen:

```python
def berechne_alter(geburtsdatum: date) -> int:
    heute = date.today()
    alter = (heute - geburtsdatum).days // 365
    if alter < 0:
        raise ValueError("Geburtsdatum liegt in der Zukunft!")
    return alter
```

Mit der Anweisung `raise ValueError(...)` wird ein 
`ValueError`-Objekt instanziiert und mit `raise` "geworfen", 
um den Fehler zu signalisieren.
Die Programmausführung wird an dieser Stelle unterbrochen, 
und die Kontrolle wird an den Aufrufer der Funktion übergeben, 
welcher die Exception behandeln kann.

*(Du hast sicher schon öfters die Erfahrung gemacht,
dass auch andere Funktionen, welche du aufrufst,
eine Exception werfen können, z.B. `datetime.strptime(...)`, 
wenn ein ungültiges Format für ein Datum übergeben wird.)*

## Behandeln ("Fangen") einer Exception

Wird eine Exception nicht behandelt,
dann fürhrt dies zum Absturz des Programms, 
und es wird eine Fehlermeldung ausgegeben, 
welche die Art der Exception und die Zeile enthält,
in welcher die Exception aufgetreten ist.

Besser ist es, die Exception zu behandeln, 
um den Fehler zu signalisieren,
ohne dass das gesamte Programm abstürzt:

In einem anderen Teil des Programms kann die Funktion 
`berechne_alter()` aus dem oben gezeigten Beispiel
in einem `try`- Block aufgerufen werden,
und die Exception kann im `except`- Block behandelt werden:

```python
try:
    alter = berechne_alter(date(2099, 1, 1))
    print(f"Alter: {alter}")
except ValueError as e:
    print(f"Fehler: {e}")
```

In diesem Beispiel wird die Funktion `berechne_alter()` 
mit einem Datum aufgerufen, welches in der Zukunft liegt, 
was eine `ValueError`- Exception auslöst.

Die Funktion `berechne_alter()` wird dann nicht bis zum 
Ende ausgeführt, sondern die Ausführung wird sofort unterbrochen,
und die Kontrolle wird an den `except`- Block übergeben,
wo die Exception behandelt wird und eine Fehlermeldung 
ausgegeben wird.

Beim `except`- Block 
kann auch der Typ der Exception angegeben werden,
um nur bestimmte Arten von Exceptions zu behandeln.
Will man alle Arten von Exceptions behandeln,
gibt man als Typ `Exception` an, 
da alle Exceptions von der Basisklasse `Exception` erben.

Im gezeigten Beispiel wird eine Referenz auf 
das `'ValueError'`- Objekt, 
welches von der Funktion `berechne_alter()` geworfen wird,
in der Variablen `e` gespeichert,
um die Fehlermeldung aus dem `Exception`- Objekt zu extrahieren.

## Ein weiteres Beispiel

Wir definieren eine Klasse `Konto`, 
welche ein Bankkonto repräsentiert. 
Darin befinden sich eine Methode `abheben()`,
und eine Methode `einzahlen()`.
Der Kontostand wird in einem Attribut `saldo` gespeichert.

```python
class Konto:
    def __init__(self, saldo: float = 0.0):
        self.saldo = saldo

    def abheben(self, betrag: float):
        if betrag > self.saldo:
            raise ValueError("Nicht genügend Guthaben!")
        self.saldo -= betrag

    def einzahlen(self, betrag: float):
        self.saldo += betrag
```

In der Methode `abheben()` wird
eine `ValueError`-Exception geworfen,
wenn der Betrag, welcher abgehoben werden soll,
größer ist als der aktuelle Kontostand (`saldo`).
Das Konto darf also nicht überzogen werden.

Dann definieren wir eine Funktion `ueberweisen`,
in welcher von einem Konto Geld auf ein anderes Konto 
überwiesen wird:

```python
def ueberweisen(von: Konto, nach: Konto, betrag: float):
        nach.einzahlen(betrag)
        von.abheben(betrag)
```

Zuerst wird der Betrag 
auf das Zielkonto eingezahlt,
und danach wird der Betrag vom Quellkonto abgehoben.
Wenn das Quellkonto nicht genügend Guthaben hat,
wird eine `ValueError`- Exception geworfen,
und das Programm stürzt ab, 
wenn die Exception nicht behandelt wird.

Im Code, welcher die Methode `ueberweisen` aufruft,
kann die Exception behandelt werden,
um den Fehler zu signalisieren,
ohne dass das gesamte Programm abstürzt:

```python
konto1 = Konto(100.0)
konto2 = Konto(50.0)
try:
    ueberweisen(konto1, konto2, 150.0)
except ValueError as e:
    print(f"Fehler: {e}")
```

Hier wird versucht, 
150.0 von `konto1` auf `konto2` zu überweisen,
was eine `ValueError`- Exception auslöst,
da `konto1` nur 100.0 Guthaben hat.
Die Exception wird im `except`- Block behandelt,
und eine Fehlermeldung wird ausgegeben,
ohne dass das gesamte Programm abstürzt.

Es bleibt aber ein Problem:
Der Betrag wurde bereits auf `konto2` eingezahlt,
bevor die Exception geworfen wurde,
was zu einem inkonsistenten Zustand führt,
da `konto2` nun ein höheres Guthaben hat,
obwohl die Überweisung nicht erfolgreich war.

Die einfachste Lösung für dieses Problem wäre es,
die Reihenfolge der Anweisungen in der Funktion `ueberweisen()`
zu ändern.

Hier soll aber gezeigt werden, was zu tun ist, 
wenn man die Reihenfolge der Anweisungen 
aus irgendeinem Grund nicht ändern kann:

```python
def ueberweisen(von: Konto, nach: Konto, betrag: float):
    nach.einzahlen(betrag)
    try:
        von.abheben(betrag)
    except ValueError as e:
        nach.abheben(betrag)  # Rückgängig machen der Einzahlung
        raise e  # Exception erneut werfen, um den Fehler zu signalisieren
```

Hier wird die Methode `abheben()`
in einem `try`- Block aufgerufen,
und die `ValueError`- Exception wird im `except`- Block behandelt.
Wenn die Exception auftritt, wird die Einzahlung auf `konto2` 
rückgängig gemacht,
indem der Betrag von `konto2` abgehoben wird.
Danach wird die Exception erneut mit `raise e` geworfen,
um den Fehler zu signalisieren,
und die Kontrolle wird an den Aufrufer der 
Funktion `ueberweisen()`
übergeben, welcher die Exception behandeln kann.

## Resourcen freigeben

In manchen Fällen ist es notwendig,
dass bestimmte Anweisungen immer ausgeführt werden,
unabhängig davon, ob eine Exception auftritt oder nicht.
In solchen Fällen kann ein `finally`- Block verwendet werden,
welcher immer ausgeführt wird,
egal ob eine Exception auftritt oder nicht.

Beispiel:

```python
try:
    datei = open("kontostaende.txt", "r")
    konto1 = Konto(float(datei.readline()))
    konto2 = Konto(float(datei.readline()))
    ueberweisen(konto1, konto2, 150.0)
except Exception as e:
    print(f"Fehler: {e}")
finally:
    datei.close()
```

In diesem Beispiel wird eine Datei geöffnet,
um die Kontostände von `konto1` und `konto2` zu lesen.
Die Methode `ueberweisen()` wird aufgerufen,
und die Exception wird im `except`- Block behandelt.

Unabhängig davon, ob eine Exception auftritt oder nicht,
wird die Datei im `finally`- Block geschlossen,
um Ressourcen freizugeben und sicherzustellen,
dass die Datei nicht offen bleibt.

## Zusammenfassung

### Wichtige Begriffe

* Exception
* try-Block
* except-Block
* finally-Block
* raise
* Exception-Objekt
* ValueError

### Das sollst du können

* Fehler in deinem Code mit Exceptions signalisieren (`raise`)
* Exceptions behandeln, um den Fehler auszugeben, 
  ohne dass das gesamte Programm abstürzt
* die Reihenfolge von Anweisungen in einem `try`-Block 
  so gestalten, dass der Code auch bei Auftreten 
  einer Exception in einem konsistenten Zustand bleibt
* Ressourcen freigeben, indem du einen `finally`-Block verwendest
* die Fehlermeldung aus einem Exception-Objekt extrahieren, 
  um sie an den Benutzer weiterzugeben
