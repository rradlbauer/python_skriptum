# 26.0 Exceptions


Oft ist es nötig, Fehler zu behandeln, 
die während der Ausführung eines Programms auftreten können.
Angenommen, du schreibst eine Funktion, 
welche von einem Geburtsdatum das Alter einer Person berechnet.
Wenn die Funktion mit einem Datum aufgerufen wird, welches in der Zukunft liegt,
wird ein negatives Alter berechnet, was keinen Sinn ergibt.
In solchen Fällen ist es sinnvoll, eine Fehlermeldung auszugeben,
um den Benutzer darauf hinzuweisen, dass etwas schiefgelaufen ist.

```python
from datetime import date
def berechne_alter(geburtsdatum: date) -> int:
    heute = date.today()
    alter = (heute - geburtsdatum).days // 365
    if alter < 0:
        print("Geburtsdatum liegt in der Zukunft!")
    return alter
```


In diesem Beispiel wird eine Fehlermeldung ausgegeben,
wenn das Geburtsdatum in der Zukunft liegt.

Das Problem bei diesem Ansatz ist, dass die Funktion `berechne_alter()`
trotzdem ein negatives Alter zurückgibt, was zu weiteren Fehlern führen kann,
wenn die Rückgabewerte der Funktion weiterverarbeitet werden.
Ein besserer Ansatz wäre es, eine Ausnahme (Exception) zu werfen,
um den Fehler zu signalisieren und die weitere Ausführung des Programms zu 
oder die Behandlung der Exception zu ermöglichen.

```python
def berechne_alter(geburtsdatum: date) -> int:
    heute = date.today()
    alter = (heute - geburtsdatum).days // 365
    if alter < 0:
        raise ValueError("Geburtsdatum liegt in der Zukunft!")
    return alter
```

Mit der Anweisung `raise ValueError(...)` wird eine `ValueError`-Objekt
instanziiert mit `raise` "geworfen", um den Fehler zu signalisieren.
Die Programmausführung wird an dieser Stelle unterbrochen, 
und die Kontrolle wird an den Aufrufer der Funktion übergeben, 
welcher die Exception behandeln kann.

In einem anderen Teil des Programms kann die Funktion `berechne_alter()` 
dann in einem `try`-Block aufgerufen werden,
und die Exception kann im `except`-Block behandelt werden:

```python
try:
    alter = berechne_alter(date(2099, 1, 1))
    print(f"Alter: {alter}")
except ValueError as e:
    print(f"Fehler: {e}")
```

In diesem Beispiel wird die Funktion `berechne_alter()` mit einem Datum aufgerufen,
welches in der Zukunft liegt, was eine `ValueError`-Exception auslöst.

Die Funktion `berechne_alter()` wird dann nicht bis zum Ende ausgeführt, 
sondern die Ausführung wird sofort unterbrochen,
und die Kontrolle wird an den `except`-Block übergeben,
wo die Exception behandelt wird und eine Fehlermeldung ausgegeben wird.
