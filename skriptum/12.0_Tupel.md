# 12.0 Tupel

Du hast schon den Datentyp `list` kennengelernt, welcher es dir ermöglicht, 
mehrere Werte in einer Variablen zu speichern.
Listen verwendet man meist, um mehrere Werte vom gleichen Typ
zu speichern, z.B. mehrere Zahlen.

Es gibt aber auch den Datentyp `tuple`, 
welcher sehr ähnlich zu `list` ist.
Tupel werden in runden Klammern geschrieben:

```python
>>> t = ("susanne", "01.01.1999", 173, 63)
>>> t
('susanne', '01.01.1999', 173, 63)
```

Wie in diesem Beispiel werden `tuple` oft verwendet,
um mehrere Werte unterschiedlichen Typs zu speichern.
In diesem Fall sind es der Name, 
das Geburtsdatum, die Größe und das Gewicht einer Person.

Ein Tupel kann auch Zahlen, Strings oder sogar andere Datenstrukturen enthalten.

## Zugriff auf Elemente

Auf einzelne Elemente eines Tupels greift man genauso zu wie bei Listen,
nämlich über den Index:

```python
>>> t[0]
'susanne'

>>> t[2]
173
```

Der Index beginnt wie bei Listen immer bei `0`.

Du kannst auch negative Indizes verwenden:

```python
>>> t[-1]
63
```

Das bedeutet: Das letzte Element des Tupels.

## Tupel sind unveränderlich

Im Gegensatz zu einer Liste kann ein Tupel **nicht verändert werden**.
Das bedeutet, dass du keine Werte hinzufügen, entfernen oder ändern kannst.

```python
>>> t[2]
173

>>> t[2] = 175
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment
```

In diesem Beispiel wird auf das dritte Element des Tupels `t` zugegriffen.
Das funktioniert genauso wie bei einer Liste.
Wenn du aber versuchst, den Wert zu ändern, erhältst du einen Fehler.

Diese Eigenschaft nennt man **Unveränderlichkeit (immutability)**.

## Wann verwendet man Tupel?

Tupel verwendet man häufig dann, wenn:

- eine Gruppe von Werten **zusammengehört**
- sich diese Werte **nicht ändern sollen**
- die Struktur **immer gleich bleibt**

Typische Beispiele:

- Koordinaten `(x, y)`
- RGB-Farben `(255, 128, 0)`
- Datensätze mit festen Eigenschaften

```python
punkt = (10, 5)
farbe = (255, 0, 0)
```

## Tupel mit einem Element

Ein Tupel mit nur einem Element benötigt ein Komma:

```python
>>> t = (5,)
>>> type(t)
<class 'tuple'>
```

Ohne Komma wäre es nur eine Zahl:

```python
>>> t = (5)
>>> type(t)
<class 'int'>
```

## Tupel entpacken

Eine sehr interessante Möglichkeit von Tupeln ist,
dass du ein Tuple einfach entpacken und dadurch
in mehrere Variablen aufteilen kannst:

```python
>>> name, geburtsdatum, groesse, gewicht = t

>>> name
'susanne'

>>> geburtsdatum
'01.01.1999'

>>> groesse
173

>>> gewicht
63
```

In diesem Beispiel wird das Tupel `t` in die Variablen 
`name`, `geburtsdatum`, `groesse` und `gewicht` aufgeteilt.

Wenn dich ein Wert nicht interessiert, 
kannst du ein Unterstrichzeichen `_` verwenden:

```python
>>> name, _, groesse, _ = t

>>> name
'susanne'

>>> groesse
173
```

Der Unterstrich `_` ist eine Konvention für:
"Dieser Wert interessiert mich nicht."

## Über Tupel iterieren

Auch über Tupel kann man mit einer `for`-Schleife iterieren.

```python
>>> t = ("susanne", "01.01.1999", 173, 63)

>>> for wert in t:
...     print(wert)

susanne
01.01.1999
173
63
```

Dabei wird jedes Element des Tupels der Reihe nach
in der Variable `wert` gespeichert.

Man kann auch mit `enumerate()` gleichzeitig
den Index und den Wert erhalten:

```python
>>> for i, wert in enumerate(t):
...     print(i, wert)

0 susanne
1 01.01.1999
2 173
3 63
```

## Listen in Tupeln

Ein Tupel kann auch eine Liste enthalten:

```python
>>> student = ("Anna", [1, 2, 2, 3])
>>> student
('Anna', [1, 2, 2, 3])
```

Hier besteht das Tupel aus:

- dem Namen
- einer Liste mit Noten

Das Tupel selbst kann nicht verändert werden,
aber die **Liste im Tupel kann verändert werden**.

```python
>>> student[1].append(1)
>>> student
('Anna', [1, 2, 2, 3, 1])
```

Warum macht man das?

Das Tupel beschreibt eine **feste Struktur eines Datensatzes**  
(Name + Notenliste).

Die Noten können sich jedoch ändern,
deshalb verwendet man dafür eine **Liste**.

## Tupel in Listen

Sehr häufig verwendet man auch **Listen von Tupeln**.

```python
>>> personen = [
...     ("Anna", 21),
...     ("Max", 19),
...     ("Julia", 23)
... ]
```

Hier ist:

- die **Liste** veränderlich
- jedes **Tupel** ein Datensatz

Zum Beispiel kann man eine neue Person hinzufügen:

```python
>>> personen.append(("Tom", 20))
```

Das Tupel selbst bleibt aber unveränderlich,
weil ein Datensatz normalerweise nicht
seine Struktur ändern soll.

Man kann darüber auch sehr gut iterieren:

```python
>>> for name, alter in personen:
...     print(name, "ist", alter, "Jahre alt")

Anna ist 21 Jahre alt
Max ist 19 Jahre alt
Julia ist 23 Jahre alt
Tom ist 20 Jahre alt
```

Hier werden die Tupel direkt in `name` und `alter`
entpackt.

## Tupel als Rückgabewerte von Funktionen

Funktionen können mehrere Werte zurückgeben,
indem sie ein Tupel zurückgeben:

```python
def min_max(liste):
    return min(liste), max(liste)

>>> min_wert, max_wert = min_max([3, 7, 2, 9])
>>> min_wert
2
>>> max_wert
9
```

Python erstellt dabei automatisch ein Tupel.
